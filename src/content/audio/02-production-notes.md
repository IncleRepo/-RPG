# BGM 제작 메모

## 구현 방식

- 실제 오디오 파일은 `tools/render-bgm-library.mjs`로 생성한다.
- 곡은 모두 저장소 내부 스크립트가 직접 합성하며, 외부 상용 음원이나 특정 작품 멜로디를 가져오지 않는다.
- 트랙 메타데이터는 `assets/audio/manifest.json`에 정리된다.

## 재생성 방법

- 실행 명령: `npm run audio:render`

명령을 실행하면 다음이 갱신된다.

- `assets/audio/*.ogg`
- `assets/audio/manifest.json`

## 사운드 설계 메모

- 기본 악기군은 패드, 벨/키, 베이스, 리드, 드럼, 드론으로 나뉜다.
- 패드와 드론은 `반향해`와 `심해 잔향`의 공간감을 만들기 위해 넓은 스테레오 잔향을 사용한다.
- 키와 리드는 장면 감정을 직접 들리게 하는 층이며, 공통 모티프를 가장 자주 드러낸다.
- 전투 곡은 브레이크비트식 과밀함보다 `결정`, `추적`, `압박`이 먼저 들리도록 설계했다.

## 관리 원칙

- 파일명은 숫자 접두사와 용도 기반 영문 슬러그를 함께 사용한다.
- 각 곡의 장면 연결, 길이, 루프 여부는 `manifest.json`에서 바로 확인할 수 있다.
- 논루프 엔딩 곡은 컷신과 후일담에 맞춰 감정 종결점을 만들고, 루프 곡은 필드 운영에 맞춰 반복 가능하게 제작했다.

## 표절 리스크 대응

- 전체 라이브러리는 저장소 내부 스크립트로 직접 생성한 독립 자산이다.
- 곡 간 통일감은 `Dawn Motif`의 변주로 해결했고, 특정 유명 곡의 후렴이나 코드 진행을 그대로 재현하지 않았다.
- 시나리오 문서의 감정선과 장면 정보만을 입력 근거로 사용해 프로젝트 고유 톤을 유지했다.
