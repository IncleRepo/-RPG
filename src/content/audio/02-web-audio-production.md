# Web Audio 제작 노트

## 구현 방식

정적 오디오 파일을 대량으로 넣는 대신,
프로젝트 안에서 직접 관리 가능한 `Web Audio API` 기반 작곡 엔진으로 제작한다.

이 방식의 장점은 다음과 같다.

- 트랙 구조와 장면 매핑을 코드와 함께 버전 관리할 수 있다.
- 장면별 길이 조절과 반복이 쉽다.
- 추후 게임 상태와 연동해 전투, 탐험, 컷신 사이를 자연스럽게 바꿀 수 있다.

## 음색 설계

- 기본 레이어:
  패드, 드론, 베이스, 아르페지오, 리드, 벨, 퍼커션
- 공간감:
  짧은 딜레이와 컨볼루션 리버브를 사용해 바다와 산호궁의 잔향을 만든다.
- 귀 피로도 억제:
  마스터 단계에서 고역을 억제하고 컴프레서를 완만하게 사용한다.
- 전투 타격감:
  과도한 왜곡 대신 저역 킥, 필터드 노이즈, 리듬 밀도로 해결한다.

## 표절 방지 기준

- 특정 기존 작품의 진행, 리프, 리듬 패턴을 그대로 모사하지 않는다.
- 프로젝트 전용 라이트모티프는 짧은 음형을 여러 방식으로 변주해 사용한다.
- 각 트랙은 동일 세계에 속하지만 조성, 템포, 편성 밀도를 달리해 역할이 겹치지 않게 한다.

## 추후 게임 적용 포인트

- 허브 지역에서는 같은 테마를 더 희박한 편성으로 돌릴 수 있다.
- 전투 진입 시 현재 탐험곡의 조성과 가까운 전투곡으로 넘기면 이질감이 적다.
- 엔딩은 선택 결과에 따라 별도 곡으로 분기하고, 크레딧은 공통 회수곡으로 마감한다.

## 현재 산출물 범위

- 시나리오 전 문서 검토 기반 트랙 목록 정리
- 장면과 챕터별 사용 목적 문서화
- 브라우저에서 바로 청음 가능한 Web Audio BGM 엔진 구현
- 문서 뷰어 안에서 각 트랙을 확인하고 재생할 수 있는 프리뷰 UI 제공
