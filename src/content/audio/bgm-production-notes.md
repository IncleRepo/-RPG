# BGM 제작 및 자산 정리 메모

## 폴더 구조

- 음원 출력: `assets/audio/bgm/`
- 기획 문서: `src/content/audio/`
- 생성 스크립트: `tools/generate-bgm.mjs`

## 생성 방식

이번 세트는 외부 유명 곡을 베끼지 않고,
프로젝트 시나리오에 맞춘 전용 모티프와 진행으로 직접 생성했다.

사용한 도구는 다음과 같다.

- `tonal`: 조성, 스케일, 음계 계산
- `wavefile`: 오프라인 렌더링용 WAV 작성
- `ffmpeg`: 최종 `.ogg` 변환

재생산 명령:

```bash
npm run generate:bgm
```

## 사운드 방향

- 고역을 세우는 대신 중역과 저역의 질감을 두껍게 쌓아 장시간 재생해도 피로가 적게 설계했다.
- 패드, 벨, 리드, 베이스, 타악을 트랙마다 조합해 챕터 개성을 분리했다.
- 공통 모티프를 여러 곡에 나눠 배치해 시나리오 흐름이 끊기지 않게 했다.
- 공간감은 짧은 다중 딜레이 기반 잔향으로 처리해 지나치게 번지지 않도록 제한했다.

## 자산 관리 기준

- 파일명은 사용 목적 순서가 보이도록 `01`부터 `10`까지 번호를 붙였다.
- 트랙 메타데이터는 `assets/audio/bgm/manifest.json`에 정리된다.
- 전체 세트 길이는 약 `13분 59초`다.
- 생성 스크립트를 함께 보관하므로 이후 길이, 조성, 편성 수정이 가능하다.

## 표절 리스크 관리 메모

- 특정 작품을 직접 참조한 리프나 진행을 넣지 않았다.
- 메인 모티프와 지역 모티프는 이번 작업에서 새로 설계했다.
- 지역별 차이를 내되, 동일 프랜차이즈 풍의 서명 멜로디를 피하고 장면 기능 중심으로 작곡했다.

## 사용 권장 방식

- 지역곡은 탐험 루프용, 전투곡은 상황 전환용으로 사용한다.
- `06`에서 `07`로 넘어갈 때는 드럼 강도와 저역 밀도 차이로 보스전 진입이 자연스럽게 느껴진다.
- `08`은 대사 장면을 덮지 않도록 가장 넓은 여백을 두고 설계했다.
- `10`은 엔딩 A/B/C 어느 분기에도 쓸 수 있도록 감정 방향을 한쪽으로 고정하지 않았다.
