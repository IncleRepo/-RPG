# 게임 UX/UI 구성 문서

## 문서 목적

이 문서는 `반향해 연대기`의 실제 플레이 화면에서
어떤 정보를 어떤 계층으로 보여 줄지 고정하는 기준안이다.

목표는 다음과 같다.

- 탐험, 전투, 대화, 이벤트, 챕터 진행에서 필요한 UI를 상태별로 분리한다.
- 프롤로그 버티컬 슬라이스 구현자가 HUD, 대화창, 목표 표시, 메뉴 구조를 바로 참조할 수 있게 한다.
- 이후 챕터 확장 시에도 같은 정보 구조를 유지하도록 화면 규칙을 정리한다.

## UX/UI 설계 목표

1. 화면은 항상 `지금 위험한 것`, `지금 할 수 있는 것`, `지금 어디로 가야 하는지`를 먼저 읽히게 해야 한다.
2. 상시 HUD는 최소화하되, 전투와 구조전처럼 압박이 커지는 순간에는 필요한 정보를 빠르게 확장해야 한다.
3. 대화와 컷신은 정보 전달보다 감정 몰입이 중요하므로 HUD를 자동 축소하거나 숨겨야 한다.
4. 챕터별 특수 규칙은 기존 HUD를 깨지 않고 `임시 상태 패널`로 덧붙이는 방식이 안전하다.
5. 키보드 기준 입력은 화면에 직접 노출해 초반 학습 부담을 줄이고, 메뉴에서는 언제든 다시 확인 가능해야 한다.

## 화면 레이어 원칙

게임 화면은 아래 다섯 레이어로 나눈다.

1. `월드 레이어`
   탐험, 전투, NPC, 기믹, 배경 연출이 보이는 실제 플레이 영역
2. `상시 HUD 레이어`
   HP, 공명, 기억 부담, 장착 스킬, 현재 목표처럼 기본 플레이 판단에 필요한 정보
3. `상황 UI 레이어`
   상호작용 프롬프트, 구조 목표, 보스 경고, 잔향 자막, 튜토리얼 카드처럼 특정 순간에만 뜨는 정보
4. `차단형 UI 레이어`
   대화창, 일시정지 메뉴, 저장/불러오기, 설정처럼 입력 우선권을 가져가는 화면
5. `전면 연출 레이어`
   컷신, 챕터 타이틀, 게임오버, 엔딩 선택처럼 플레이를 잠시 완전히 전환하는 화면

## 플레이 상태별 UI 사용 시점

| 플레이 상태   | 진입 조건                                     | 항상 보이는 UI                      | 일시적으로 보이는 UI                              | 숨겨야 하는 UI                                      |
| ------------- | --------------------------------------------- | ----------------------------------- | ------------------------------------------------- | --------------------------------------------------- |
| 탐험          | 자유 이동, 조사, 플랫폼 이동                  | 상시 HUD, 현재 목표 축소형          | 상호작용 프롬프트, 튜토리얼 키 힌트, 지역명 배너  | 대화 선택지, 저장 목록                              |
| 조사/상호작용 | 오브젝트 조사, 기록기 복구, 닻 포인트 사용 전 | 상시 HUD 축소형                     | 상호작용 카드, 조사 결과 로그, 보상 토스트        | 전투 경고 패널                                      |
| 대화          | NPC 대화, 잔향 청취, 메인 스토리 대사         | HP/공명 최소 표시 또는 숨김         | 대화창, 초상/화자명, 선택지, 잔향 자막            | 스킬 바의 강조 효과, 구조전 타이머 외 불필요 토스트 |
| 전투          | 일반 적 교전, 방어전, 보스전                  | 상시 HUD 전체, 적 정보, 목표 패널   | 피격 수치, 붕괴 경고, 회피 타이밍 알림, 보스 경고 | 긴 조사 로그, 튜토리얼 카드가 아닌 일반 설명 팝업   |
| 컷신/연출     | 주요 진실 공개, 챕터 전환, 카메라 강제 연출   | 필요 시 목표 한 줄 또는 자막만 유지 | 자막, 연출용 이름표, 챕터 타이틀                  | 상시 HUD 대부분                                     |
| 메뉴/일시정지 | ESC, 옵션 버튼, 강제 일시정지                 | 흐릿한 배경 위 메뉴 패널            | 인벤토리, 장비, 스킬, 설정, 저장/불러오기         | 전투 피격 이펙트, 조작 힌트                         |
| 결과/실패     | 구조전 종료, 보스 승리, 게임오버              | 결과 전용 패널                      | 보상 요약, 손실 재화, 재시도 선택                 | 기존 HUD 전체                                       |

## 기본 화면 배치

### 좌측 상단: 생존 및 상태 묶음

- 플레이어 초상 또는 간단한 실루엣
- HP 바
- 공명 바
- 기억 부담 단계 표시
- 현재 적용 중인 주요 상태 이상 또는 보호 효과 최대 3개

이 묶음은 플레이어 생존과 리스크 판단을 담당한다.
전투 시에는 수치와 애니메이션을 강조하고,
탐험 시에는 약간 축소된 밀도로 유지한다.

### 우측 상단: 목표 및 지역 정보 묶음

- 현재 메인 목표 1줄
- 필요 시 서브 목표 1~2개
- 구조전/방어전 시 남은 대상 수 또는 방어 대상 내구도
- 새 지역 진입 시 2초 내외의 지역명 배너

항상 미니맵을 두기보다
현재 목표와 구역명으로 방향성을 주는 쪽이 이 프로젝트에 더 맞다.

### 하단 우측: 액션 및 스킬 묶음

- 기본 공격 아이콘은 두지 않고 입력 힌트만 필요 시 노출
- 액티브 스킬 3종
- 동행 지원 스킬 1종
- 종편 공명 스킬 1종
- 각 슬롯의 키 라벨, 쿨다운, 공명 소모 색상, 사용 불가 사유

스킬 바는 탐험 중에는 얇게,
전투 돌입 시에는 배경 패널과 쿨다운 시계가 더 또렷하게 보이도록 전환한다.

### 하단 좌측: 대화 및 로그 묶음

- 일반 탐험 중에는 최근 조사 로그 한 줄만 짧게 노출
- 대화 진입 시 전체 대화창 영역으로 확장
- 잔향 음성이나 시스템 메시지는 대화창과 다른 색으로 구분

## 상시 HUD 상세 규칙

### HP 표시 방식

- 기본 형태는 가로 바 1줄
- 최근 피해 구간은 짙은 잔상으로 0.6초 정도 남겨 즉시 피해량을 읽게 한다.
- 회복은 밝은 색 채움이 뒤에서 따라붙는 방식으로 처리한다.
- 최대 HP 감소 상태가 있으면 바의 끝 일부를 어둡게 막아 기억 부담의 체감이 바로 오게 한다.

### 공명 표시 방식

- HP 아래 보조 바 1줄로 배치한다.
- 스킬 사용 가능 기준선이 보이게 얇은 눈금을 둔다.
- 전투 후 자동 회복이 발생할 때는 별도 팝업보다 바 자체의 파형 애니메이션으로 표현한다.

### 기억 부담 표시 방식

- 수치보다 `단계형`이 중요하므로 3단계 아이콘 또는 고리 형태가 적합하다.
- 낮음: 희미한 금 균열
- 중간: 색이 짙어지고 HUD 외곽에 약한 떨림
- 높음: 아이콘 경고, 화면 가장자리 왜곡, HP 최대치 축소 표시

이 값은 화면 중앙 경고보다
상시 보이는 작은 불안감으로 유지하는 편이 세계관과 잘 맞는다.

### 상태 이상 및 보호 효과 표시

- 좌측 상단 상태 묶음 하단에 작은 아이콘으로 노출
- 최대 3개까지만 상시 노출
- 4개 이상이면 `+n`으로 묶고 상세는 일시정지 메뉴에서 확인

## 액션 및 스킬 UI

### 기본 액션 노출 방식

기본 공격, 점프, 회피는 상시 버튼 아이콘으로 고정하지 않는다.
대신 아래 방식으로 노출한다.

- 초반 튜토리얼 구간에서 키 힌트 카드 1회 노출
- 특정 상황에서 실패 가능성이 높을 때만 화면 가장자리 근처에 짧게 재노출
- 메뉴의 `조작 가이드` 탭에서 항상 재확인 가능

상시 HUD에 모든 조작을 넣으면 화면이 과밀해지므로,
전투 핵심 자원과 스킬에 우선권을 준다.

### 스킬 슬롯 구성

| 슬롯       | 역할                  | 표시 정보                                               |
| ---------- | --------------------- | ------------------------------------------------------- |
| 액티브 1~3 | 공격, 제어, 생존 스킬 | 아이콘, 키 라벨, 쿨다운, 공명 소모                      |
| 지원 스킬  | 동행 호출             | 동료 초상 소형 아이콘, 대기 상태, 사용 가능 반짝임      |
| 종편 공명  | 강한 결단형 스킬      | 별도 프레임, 기억 부담 증가 경고, 사용 직후 경계 이펙트 |

### 사용 불가 피드백

- 공명 부족: 파란 계열 비활성 + `공명 부족`
- 쿨다운 중: 원형 또는 부채꼴 타이머
- 특정 스토리 봉인 상태: 회색 잠금 + 짧은 설명
- 기억 부담 고단계로 제약 발생 시: 슬롯 테두리에 금이 간 표현

### 보스전 강조 규칙

- 보스 진입 시 스킬 바 배경을 조금 더 짙게 만들어 시인성을 높인다.
- 종편 공명 슬롯은 붕괴 상태 유도 타이밍에서 약한 펄스를 줘 선택지를 상기시킨다.
- 단, `자동 추천`처럼 보이면 안 되므로 입력 강요 수준의 깜빡임은 피한다.

## 상호작용 가능 오브젝트 표시 방식

### 기본 원칙

- 상호작용 가능 여부는 `가까이 갔을 때만` 분명하게 보이게 한다.
- 멀리서 모든 오브젝트에 아이콘을 띄우지 않는다.
- 중요한 메인 목표 오브젝트만 화면 바깥 가장자리 방향 표시를 허용한다.

### 프롬프트 구성

- 오브젝트 위 또는 옆: 작은 강조 테두리
- 플레이어 근접 시: `E 조사`, `E 대화`, `E 활성화` 같은 키 포함 프롬프트
- 희귀 또는 메인 오브젝트: 배경색을 달리해 중요도를 구분

### 오브젝트 종류별 차이

| 오브젝트         | 표시 방식                                            |
| ---------------- | ---------------------------------------------------- |
| NPC              | 머리 위 말풍선 점 또는 잔향 파형                     |
| 조사 오브젝트    | 얇은 윤곽선 + `조사` 프롬프트                        |
| 기록기/기믹 장치 | 공명 파동형 하이라이트 + `활성화`                    |
| 닻 포인트        | 멀리서도 보이는 수직 빛기둥, 가까이서 기능 요약 패널 |
| 루팅 오브젝트    | 짧은 반짝임 후 접근 시 `획득` 프롬프트               |

### 새벽 청취 전용 표시

- 일반 대화와 다른 잔향 파형 아이콘 사용
- 사용 가능 시 오브젝트 주변 공기가 흔들리는 듯한 반투명 오라
- 청취 중에는 화면 좌우 가장자리에 얇은 청음선 이펙트를 넣어 `지금 듣고 있는 정보가 현재 대화와 다르다`는 점을 전달

## 대화 UI 구조

### 기본 대화창 배치

- 화면 하단 25~30% 높이
- 화자명
- 대사 본문
- 필요 시 초상 또는 인물 실루엣
- 다음 넘김, 자동 재생, 로그 열람, 스킵 표시

### 대화 종류별 구분

| 대화 종류          | 시각 규칙                                   |
| ------------------ | ------------------------------------------- |
| 일반 NPC 대화      | 기본 패널, 화자명 강조                      |
| 메인 시나리오 대화 | 배경 패널 강화, 초상 또는 실루엣 사용       |
| 새벽 청취 / 잔향   | 다른 색의 자막 레이어, 현재 화자와 분리     |
| 무전 / 기록 재생   | 화자명 대신 장치명 표시, 파형 장식 사용     |
| 백야 개입 대사     | 패널보다 화면 중앙 또는 인접 위치 자막 우선 |

### 선택지 구조

- 평시에는 2~3개 선택지만 사용
- 중요한 감정 선택 또는 최종 엔딩 선택만 별도 전면 선택 UI로 확장
- 선택지는 대화창 바로 위에 쌓지 말고, 대화창 내부 하단에 명확히 분리해 배치한다.

### 대화 중 HUD 처리

- 일반 NPC 대화: HP/공명만 희미하게 남기고 스킬 바는 숨긴다.
- 메인 컷신 대화: HUD 대부분 숨김, 자막과 대화창만 유지
- 전투 중 짧은 트리거 대사: 대화창 전체를 띄우지 않고 인물명 + 한 줄 대사만 상단 또는 캐릭터 근처에 표시

## 현재 목표 / 퀘스트 표시 방식

### 상시 목표 표시

- 우측 상단에 메인 목표 1개를 고정
- 서브 목표는 최대 2개까지 접힌 목록으로 노출
- 숫자 진행형 목표는 `파문 기록기 2/3`, `구조 대상 4/6`처럼 즉시 읽히는 형식 사용

### 갱신 연출

- 목표가 바뀌면 화면 중앙 상단에 2초 내외 배너 노출
- 완료 시 작은 체크와 함께 우측 상단 축소 목록이 갱신
- 목표 변경과 동시에 관련 오브젝트나 다음 구역에 짧은 방향 힌트를 줄 수 있다.

### 퀘스트 종류별 표현

| 종류        | 화면 처리                                      |
| ----------- | ---------------------------------------------- |
| 메인 퀘스트 | 우측 상단 고정                                 |
| 지역 퀘스트 | 메뉴의 퀘스트 탭 중심, 필요 시 축소 노출       |
| 정서 퀘스트 | 대화 후 기록 갱신 중심, HUD 고정 노출은 최소화 |

### 챕터 특수 목표 패널

아래 목표는 일반 목표와 분리된 임시 패널이 필요하다.

- 구조전: 남은 생존자 수, 제한 시간, 방어 대상 상태
- 챕터 2 시민 보호전: 시민 구조 수와 코어 안정도 동시 표시
- 챕터 3 감정 추적 구간: 활성 잔향 대상과 미확인 기록 수
- 엔딩 직전: 선택 불가 경고가 아닌 `결정 전 대화 가능 인물` 안내

## 일시정지 / 메뉴 / 설정 화면 구조

### 일시정지 메인 구조

`ESC` 입력 시 아래 6개 탭을 가진 오버레이 메뉴를 연다.

1. 상태
2. 장비
3. 스킬
4. 퀘스트/기록
5. 저장/불러오기
6. 설정

좌측은 탭 목록,
중앙은 현재 선택한 상세 패널,
우측은 플레이어 요약 정보로 구성하면 안정적이다.

### 상태 탭

- 현재 HP, 공명, 기억 부담 단계
- 인양 등급
- 공명 노드 투자 현황
- 장착 중인 핵심 패시브 요약

### 장비 탭

- 주무기 모듈
- 공명 장치
- 항해 부적
- 비교 수치보다 `역할 설명`이 먼저 보이게 구성

### 스킬 탭

- 장착 액티브 3종
- 지원 스킬 1종
- 종편 공명 1종
- 교체 가능 목록
- 스킬 상세 설명, 공명 소모, 쿨다운, 추천 역할

### 퀘스트/기록 탭

- 메인 퀘스트
- 지역 퀘스트
- 정서 퀘스트
- 회수한 기록 문서와 파문 로그

이 탭은 단순 할 일 목록보다
서사 이해를 돕는 기록실 성격을 함께 가져야 한다.

### 설정 탭

- 화면: 해상도, 화면 흔들림, 자막 크기
- 사운드: BGM, 효과음, 대사 음량
- 입력: 키 재설정, 조작 가이드
- 접근성: 대화 자동 재생 속도, 잔향 자막 구분 강도, 고대비 모드

## 저장 / 불러오기 UI

### 진입 위치

- 허브 선실
- 닻 포인트
- 일시정지 메뉴의 저장/불러오기 탭

### 슬롯 구성

- 자동 저장 1칸
- 수동 저장 3칸
- 각 슬롯에는 챕터명, 지역명, 플레이 시간, 마지막 목표를 표시

### 저장 화면에 반드시 필요한 정보

- 챕터 진행도
- 현재 지역
- 플레이 시간
- 인양 등급
- 종편 회수 개수
- 저장 시각

### 닻 포인트 전용 UI

닻 포인트 상호작용 시 전체 메뉴를 바로 열지 않고
먼저 짧은 허브 패널을 띄우는 편이 좋다.

- 휴식
- 저장
- 장비 변경
- 스킬 재장착
- 빠른 이동
- 나가기

이 구조면 필드 리듬을 끊지 않으면서도
닻 포인트가 안전 구역이라는 인상을 줄 수 있다.

## 게임오버 / 결과 화면 구성

### 일반 게임오버

- 중앙에 실패 원인 한 줄
- 복귀 체크포인트 위치
- 손실 재화 요약
- 기억 부담 증가 여부
- `재시도`, `최근 저장 불러오기`, `타이틀로` 선택

중요한 점은 패널이 처벌보다
`무엇이 유지되고 무엇이 줄었는지`를 명확히 설명해야 한다는 것이다.

### 구조전 / 방어전 결과

구조전은 무조건 게임오버보다 `결과 화면`이 더 적합하다.

- 구조 성공 인원
- 추가 보상 여부
- 잃은 지원 또는 후속 대사 변화 요약
- 다음 목표 버튼

### 보스전 승리 결과

- 획득 보상
- 종편 회수 여부
- 새 해금 요소
- 다음 목적지 또는 컷신 진입 버튼

## 컷신 및 연출 중 HUD 숨김 규칙

### 완전 숨김

- 챕터 오프닝
- 새벽종 파손 확인 같은 핵심 진실 공개
- 엔딩 선택 직전 전면 연출

### 최소 유지

- 추격 연출처럼 실제 입력이 계속 필요한 장면
- 방파제 탈출처럼 실패 가능성이 남아 있는 시퀀스

이 경우 남겨야 하는 것은 아래 정도면 충분하다.

- HP
- 현재 즉시 행동 키
- 실패와 직접 연결된 목표 패널

### 자동 복귀

- 컷신 종료 후 HUD는 0.2~0.4초 정도 딜레이를 두고 순차 복귀
- 목표 패널은 컷신에서 바뀐 정보가 있으면 마지막에 갱신 애니메이션과 함께 등장

## 키보드 기준 조작 UI

### 권장 기본 배치

| 동작                 | 권장 키             |
| -------------------- | ------------------- |
| 좌우 이동            | `A` `D` 또는 방향키 |
| 점프                 | `Space`             |
| 회피                 | `Left Shift`        |
| 상호작용 / 대화 진행 | `E`                 |
| 기본 공격            | `J`                 |
| 액티브 스킬 1        | `K`                 |
| 액티브 스킬 2        | `L`                 |
| 액티브 스킬 3        | `I`                 |
| 동행 지원 스킬       | `O`                 |
| 종편 공명 스킬       | `U`                 |
| 일시정지             | `Esc`               |

이 배치는 권장안이며,
설정에서 재배치 가능해야 한다.

### 화면 내 조작 가이드 원칙

- 탐험 초반에는 `이동`, `점프`, `상호작용`만 단계적으로 노출
- 전투 첫 진입 시 `기본 공격`, `회피`, `스킬 슬롯` 노출
- 새 스킬 해금 시 해당 키와 기능을 짧은 카드로 재노출
- 메뉴의 `조작 가이드`에서 전체 키 배치를 다시 볼 수 있어야 함

## 모바일 대응 시 고려할 UI 요소

이번 이슈 범위는 키보드 기준이지만,
모바일 대응을 염두에 두고 아래 기준은 미리 남겨 둔다.

- 좌측 하단 가상 이동 패드
- 우측 하단 액션 묶음: 공격, 점프, 회피, 스킬
- 상호작용은 오브젝트 근접 시 별도 큰 버튼으로 확장
- 대화창 버튼은 모바일에서 오입력 방지를 위해 하단 전체 폭 기준으로 넓게 잡는다.
- 메뉴는 다단 탭보다 1열 목록 + 전체 폭 상세 패널 구조가 유리하다.

모바일에서는 상시 목표와 상태 아이콘 크기를 더 키워야 하므로,
항상형 HUD의 정보 밀도는 PC보다 한 단계 낮추는 편이 안전하다.

## 프롤로그 버티컬 슬라이스 적용 기준

### PRO-01 오프닝 환시와 조작 인계

- 자막 중심
- 목표 배너 1회 노출
- 이동 키 힌트 1회
- HUD는 조작 인계 직후에만 등장

### PRO-02 항구 초입 이동 튜토리얼

- 상호작용 프롬프트 첫 노출
- 조사 로그 한 줄 피드백
- 점프 키 힌트 1회

### PRO-03 멈춘 어시장 조사

- 목표 패널 `기록기 0/3`
- NPC 대화창과 잔향 자막 동시 운용
- 새벽 청취 가능 오브젝트 강조

### PRO-04 시계탑 입구 개방

- 새 목표 갱신 배너
- 닻 포인트 기능 패널
- 자동 저장 알림

### PRO-05 이후 전투 입문

- 스킬 바 전체 표시 시작
- 적 HP 및 균열 게이지 노출
- 전투 종료 후 보상 토스트와 다음 목표 갱신

### PRO-08 붕괴 탈출과 PRO-09 보스전

- 탈출 구간은 HP와 즉시 목표만 남기는 축소 HUD
- 보스전은 상시 HUD 전체 + 보스 체력/균열 패널 + 위험 경고
- 컷신 진입 시 HUD 숨김, 종료 후 순차 복귀

## 후속 구현 체크리스트

- 상시 HUD, 상황 UI, 차단형 UI를 컴포넌트 레벨에서 분리할 것
- 목표 시스템은 메인/서브/임시 특수 목표를 같은 데이터 구조로 관리할 것
- 대화 UI는 일반 대화와 잔향 대사를 같은 프레임워크 안에서 다른 테마로 처리할 것
- 닻 포인트는 저장, 회복, 장비, 빠른 이동의 공통 허브 UI로 설계할 것
- 구조전과 보스전은 별도 UI를 새로 만들기보다 임시 패널만 추가하는 확장 구조를 우선할 것

## 문서 최종 결론

이 게임의 UX/UI는 `많은 정보를 항상 띄우는 RPG HUD`보다
`플레이 상태에 따라 정보량이 분명히 전환되는 스토리형 액션 RPG UI`가 적합하다.

정리하면 다음 기준으로 고정한다.

- 탐험 중에는 `상시 HUD 최소화 + 목표와 상호작용 강조`
- 전투 중에는 `생존, 자원, 스킬, 적 상태`를 빠르게 읽히게 확장
- 대화와 컷신 중에는 `감정 몰입을 위해 HUD 축소 또는 숨김`
- 메뉴와 저장 UI는 `닻 포인트`를 중심 허브로 묶어 관리
- 프롤로그부터 엔딩까지 같은 정보 구조를 유지하되 챕터 특수 규칙은 임시 패널로 확장
