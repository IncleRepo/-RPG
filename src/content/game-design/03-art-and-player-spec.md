# 아트 기준 및 플레이어 스펙 문서

## 문서 목적

이 문서는 `반향해 연대기`의 버티컬 슬라이스와 초기 양산 단계에서
아트, 맵, 충돌, 카메라, UI가 같은 기준으로 움직이도록
최소 비주얼 스펙을 고정하는 기준안이다.

목표는 다음과 같다.

- 구현자와 도트 작업자가 같은 `픽셀 단위`를 쓰게 한다.
- 플레이어, NPC, 적, 오브젝트의 비율 차이를 수치로 고정한다.
- 프롤로그 버티컬 슬라이스에 필요한 최소 모션과 에셋 범위를 정한다.
- 이후 챕터 확장 시에도 화면 구도와 UI 비율이 흔들리지 않게 한다.

## 1. 화면 기준 해상도 방향

### 논리 해상도

- 월드 렌더링 기준 해상도는 `320 x 180 px`로 고정한다.
- 실제 출력은 `x3`, `x4`, `x5` 정수 배율을 우선 사용한다.
- 기본 개발 기준 해상도는 `1280 x 720 px`로 본다.
- 가로 진행이 중심인 게임이므로 `16:9` 비율을 유지한다.

### 해상도 선택 이유

- `320 x 180`은 2D 횡스크롤에서 화면 구도를 읽기 쉽고,
  `16 px` 타일과 조합했을 때 가로 `20 타일`이 정확히 들어간다.
- 브라우저 환경에서도 픽셀 스냅을 유지하기 쉽다.
- 추후 모바일 대응 시에도 좌우 시야 우선 구도를 유지하기 좋다.

### 화면 안전 영역

- 상시 HUD 안전 여백은 기준 해상도에서 사방 `12 px` 이상 확보한다.
- 컷신 자막과 목표 배너는 상단 `16 px` 이내에 붙이지 않는다.
- 하단 대화창 최대 높이는 `56 px`를 넘기지 않는다.
- 전투 중 플레이어와 적 핵심 실루엣은 중앙 `320 x 124 px` 영역 안에서 읽혀야 한다.

## 2. 월드 타일과 좌표 기준

### 기본 타일

- 월드 기본 타일은 `16 x 16 px` 1종으로 통일한다.
- 반 타일 단위 작업이 필요하면 `8 x 8 px`를 보조 단위로 쓴다.
- 충돌은 가능하면 `16 px` 단위, 세밀 조정은 `8 px` 단위까지만 허용한다.

### 타일 사용 원칙

- 지면, 벽, 플랫폼, 사다리, 난간, 문틀은 `16 px` 격자 기준으로 배치한다.
- 정밀 플랫폼 액션이 아닌 탐험형 횡스크롤이므로 `1~2 px` 단위 미세 발판은 금지한다.
- 한 화면 안에서 바닥 기준선은 가능한 `16 px` 격자에 맞춘다.
- 장식 오브젝트가 격자를 일부 넘더라도 발판과 충돌 박스는 격자를 유지한다.

### 권장 구조 치수

| 요소                        | 권장 크기                    |
| --------------------------- | ---------------------------- |
| 기본 바닥 두께              | `1 타일`                     |
| 낮은 턱                     | `0.5 타일`                   |
| 점프로 자연스럽게 넘는 박스 | 높이 `1 타일`, 폭 `1~2 타일` |
| 짧은 단차                   | 높이 `2 타일`                |
| 문/입구 폭                  | `2 타일`                     |
| 문/입구 높이                | `3 타일`                     |
| 체크포인트 장치 폭          | `2 타일`                     |
| 체크포인트 장치 높이        | `3~4 타일`                   |

## 3. 플레이어 기본 스펙

### 플레이어 비주얼 / 충돌 기준

| 항목              | 값                               |
| ----------------- | -------------------------------- |
| 스프라이트 캔버스 | `24 x 40 px`                     |
| 실루엣 실제 점유  | 가로 `20~22 px`, 세로 `38~40 px` |
| 기본 충돌 박스    | `12 x 30 px`                     |
| 발 위치 기준점    | 스프라이트 하단 중앙             |
| 머리 여백         | 스프라이트 상단 `2~3 px` 허용    |
| 좌우 여백         | 각 `1~2 px` 허용                 |

### 비율 해석

- 플레이어는 `16 px` 타일 기준 `1.5 타일` 폭, `2.5 타일` 높이로 본다.
- 정면이 아니라 측면 실루엣이 중심이므로
  읽히는 포인트는 `외투 자락`, `벨트 장비`, `상체 기울기`다.
- 머리 비율은 전체 높이의 `25~28%` 선에서 유지한다.
- 무기보다 외투와 등 장비가 먼저 읽히는 실루엣을 우선한다.

### 충돌 / 판정 기준

| 판정                     | 기준                          |
| ------------------------ | ----------------------------- |
| 서기 충돌                | `12 x 30 px`                  |
| 앉기 미사용              | 초기 범위에서는 별도 없음     |
| 상호작용 거리            | 전방 `14 px`, 상하 `12 px`    |
| 기본 공격 유효 전방 거리 | `18~24 px`                    |
| 근접 피격 기준폭         | 플레이어 중심에서 좌우 `8 px` |
| 발판 허용 오차           | 하강 시 `4 px`                |

### 이동 기준

프롤로그 구현 기준 권장값은 다음과 같다.

| 항목                | 권장값          |
| ------------------- | --------------- |
| 걷기 속도           | 초당 `72~84 px` |
| 전투 이동 속도      | 초당 `88~96 px` |
| 점프 최고 높이      | `40~48 px`      |
| 점프 수평 도달 거리 | `48~64 px`      |
| 회피 거리           | `28~36 px`      |
| 착지 후 제어 회복   | `0.08~0.12초`   |

이 값은 플레이어가 `1~2 타일` 단차를 답답하지 않게 넘으면서도
정밀 플랫포머처럼 보이지 않게 만드는 범위다.

## 4. NPC / 적 / 오브젝트 비율 기준

### 캐릭터 비율 테이블

| 분류             | 권장 스프라이트        | 플레이어 대비 비율 | 비고                 |
| ---------------- | ---------------------- | ------------------ | -------------------- |
| 일반 NPC         | `22 x 38 px`           | `0.95x`            | 시민, 상인, 연구자   |
| 동행 주요 NPC    | `24 x 40 px`           | `1.0x`             | 사야 렌 등 주요 동료 |
| 중장비형 NPC     | `26 x 42 px`           | `1.05x`            | 라오 템 같은 체격형  |
| 일반 소형 적     | `20 x 20 ~ 24 x 24 px` | 높이 `0.5~0.6x`    | 균열 생물, 드론형 적 |
| 일반 인간형 적   | `24 x 40 px`           | `1.0x`             | 근접 병사, 추적자    |
| 엘리트 적        | `32 x 48 px`           | `1.2x`             | 미니 보스            |
| 대형 적          | `48 x 56 ~ 48 x 64 px` | `1.5~1.6x`         | 파수체, 대형 수호기  |
| 보스급 핵심 부위 | 최소 높이 `64 px` 이상 | `1.6x+`            | 화면 고정 연출 전제  |

### 오브젝트 비율 테이블

| 분류               | 권장 크기                      | 기준                   |
| ------------------ | ------------------------------ | ---------------------- |
| 조사 오브젝트 소형 | `16 x 16 px`                   | 단독 아이템, 파편      |
| 조사 오브젝트 중형 | `16 x 32 px` 또는 `32 x 32 px` | 상자, 기록기, 레버     |
| 핵심 상호작용 장치 | `32 x 48 px`                   | 닻 포인트, 기록 장치   |
| 엄폐 / 장애물      | `32 x 16 px`, `32 x 32 px`     | 수레, 잔해             |
| 배경 문 / 게이트   | `32 x 48 px` 이상              | 출입구, 봉인문         |
| 배경 랜드마크      | `64 x 64 px` 이상              | 종탑 장치, 광장 구조물 |

### 비율 적용 원칙

- 일반 시민 NPC는 플레이어보다 작아 보이게 하지 않는다.
- 주요 동료는 플레이어와 같은 높이 계열로 맞추고,
  실루엣 차이는 장비와 자세로 만든다.
- 엘리트 적은 최소 `1.2배` 이상이어야 전투 구역 진입 시 위협도가 읽힌다.
- 상호작용 장치는 플레이어 높이의 `0.75~1.2배` 범위 안에 두어
  지나치게 거대해 보이지 않게 한다.

## 5. 카메라 거리와 화면 구성 방향

### 기본 카메라

- 기본 시야는 `320 x 180 px`, 즉 가로 `20 타일`을 보여 준다.
- 플레이어는 화면 정중앙 고정보다 `진행 방향 쪽 38~42%` 지점에 둔다.
- 수직 위치는 화면 하단에서 `38~42%` 높이에 놓아 바닥 읽기를 우선한다.
- 상하 시프트는 크게 쓰지 말고, 점프나 낙하 시 `8~12 px`만 부드럽게 보정한다.

### 전투 카메라

- 일반 교전에서는 카메라를 크게 흔들지 않는다.
- 엘리트전과 보스전은 최대 `110%`까지 완만하게 줌아웃할 수 있다.
- 전투 카메라 확대 범위는 `352 x 198 px` 이내로 제한한다.
- 적이 화면 밖에서 공격하지 않도록, 전투 구역 폭은 기본 시야의 `1.2배` 이내로 잡는다.

### 프롤로그 화면 구성 기준

| 구분           | 기준                                          |
| -------------- | --------------------------------------------- |
| 안전 구역      | 플레이어 전방 `8~10 타일` 확보                |
| 일반 이동 구역 | 유효 진행 폭 `14~18 타일`                     |
| 조사 구역      | 핵심 조사 오브젝트 1개당 가로 `3~4 타일` 여유 |
| 전투 구역      | 적 2~3체 기준 가로 `10~14 타일`               |
| 보스 구역      | 최소 가로 `18 타일`, 세로 `8 타일`            |

### 카메라 연출 원칙

- 플랫폼 난도보다 `다음 정보 예고`가 중요하므로 전방 시야를 넉넉하게 둔다.
- 대화와 컷신에서도 완전한 클로즈업보다
  배경 구조물이 함께 읽히는 `허리샷~전신샷` 중심이 적합하다.
- 새벽종, 기록기, 닻 포인트는 카메라가 잠깐 멈춰도
  플레이어와 같은 화면 안에 들어오게 배치한다.

### 모션 제작 원칙

- 프롤로그 단계에서는 `12~15 FPS` 체감으로 충분하다.

- 픽셀 수를 늘리는 대신 포즈 차이를 크게 둬 실루엣으로 읽히게 만든다.

## 7. 기본 공격 / 피격 / 이동 연출 방향

### 이동 연출

- 기본 이동은 가볍게 미끄러지는 느낌보다 `체중이 앞에 실리는 항해자 보행`이 맞다.
- 가속과 감속은 짧게 두고, 브라우저 조작에서 둔하게 느껴지지 않게 한다.
- 점프는 액션 게임식 고공 도약보다 `짧고 기능적인 점프`로 보이게 한다.

### 공격 연출

- 플레이어의 기본 공격은 칼날보다 `공명 잔광이 따라붙는 근접 참격` 쪽이 적합하다.
- 1타는 빠른 확인, 2타는 전진성, 3타는 넉백을 명확히 구분한다.
- 공격 이펙트 전방 길이는 플레이어 폭의 `0.8~1.2배`를 넘기지 않는다.
- 강공격은 예고 동작 `1프레임`이라도 분명히 보여 실수 입력처럼 보이지 않게 한다.

### 피격 연출

- 소형 피격은 `1~2 px` 후퇴와 상체 젖힘으로 충분하다.
- 큰 피격만 과장된 넉백을 사용하고, 일반 적 교전에서는 과도한 튕김을 피한다.
- 붉은 피 대신 `유리 파편`, `금빛 균열`, `청록 잔광` 계열 이펙트를 우선 사용한다.

### 공명 능력 연출

- `선행 발자국`: 본체 잔상 1개 + 착지 지점 작은 균열 원형 이펙트
- `미도착 상처`: 몸 주변 얇은 금선 보호막 + 피격 시 지연 균열 파편
- `새벽 청취`: 화면 전면 이펙트보다 오브젝트 주변 공기 왜곡과 자막 레이어 차이로 표현

## 8. 월드 오브젝트와 UI 비율 기준

### 월드 대 UI 스케일 원칙

- 월드 기본 단위는 `16 px`, UI 기본 단위는 `8 px` 배수로 설계한다.
- 월드 타일 1칸과 같은 `16 px` 아이콘을 UI에 그대로 쓰지 않는다.
- UI는 월드보다 한 단계 선명하게 읽혀야 하므로
  주요 아이콘은 `16 px`, 핵심 슬롯은 `20~24 px`로 설계한다.

### HUD 권장 치수

| UI 요소               | 권장 크기                      |
| --------------------- | ------------------------------ |
| HP / 공명 바 높이     | `6~8 px`                       |
| 상태 아이콘           | `12 x 12 px`                   |
| 스킬 슬롯             | `20 x 20 px` 또는 `24 x 24 px` |
| 상호작용 키 배지      | 높이 `14~16 px`                |
| 목표 배너 높이        | `18~20 px`                     |
| 대화창 본문 영역 높이 | `40~56 px`                     |

### 배치 원칙

- UI는 월드 오브젝트를 가리지 않게 가장자리 위주로 배치한다.
- 상호작용 프롬프트는 오브젝트 상단 `8~12 px` 범위에 띄운다.
- 보스명과 목표 배너는 상단 `20 px` 안쪽에서 해결하고
  전면 경고 패널을 남발하지 않는다.

## 11. 최종 고정안

초기 구현에서는 아래 기준을 우선 고정한다.

1. 월드는 `16 x 16 px` 타일 기준으로 제작한다.
2. 플레이어는 `24 x 40 px`, 충돌은 `12 x 30 px`로 잡는다.
3. 화면은 `320 x 180 px`, 기본 시야는 `20 타일` 가로폭을 유지한다.
4. 프롤로그 필수 모션은 `15종`을 최소 기준으로 본다.
5. NPC와 인간형 적은 플레이어와 같은 높이 계열, 엘리트는 최소 `1.2배` 이상으로 잡는다.
6. UI는 월드보다 한 단계 큰 `20~24 px` 슬롯 기준으로 읽히게 만든다.
7. 버티컬 슬라이스 아트 제작은 `플레이어 -> 항구/시계탑 타일셋 -> 조사 오브젝트 -> 적/보스 -> HUD` 순서로 진행한다.

이 문서를 이후 도트, 맵, 충돌, 카메라 구현의 공통 기준으로 사용한다.
