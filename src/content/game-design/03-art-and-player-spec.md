## 3. 화면과 그리드

### 화면 규칙

- 기준 제작 해상도와 기본 검수 해상도는 `1280 x 720 px`로 고정한다.
- 월드 내부 렌더링은 `320 x 180 px`를 사용한다.
- 출력은 기본 `x4` 배율을 사용한다.
- 다른 배율 실험은 가능하지만, 아트 작업 기준은 `720p`를 우선한다.

### 안전 영역

- HUD 안전 여백: 사방 `12 px` 이상
- 상단 배너 최소 여백: `16 px`
- 하단 대화창 최대 높이: `56 px`
- 중앙 `320 x 124 px` 영역에서 플레이어와 적 핵심 실루엣이 읽혀야 한다.

### 그리드 규칙

- 충돌, 발판, 상호작용 기준은 `16 x 16 px`를 사용한다.
- 세밀 조정은 `8 x 8 px`까지만 허용한다.
- `1~2 px` 단위 발판 설계는 금지한다.

## 4. 월드 아트 제작 단위

| 분류          | 기본 캔버스                | 용도                      |
| ------------- | -------------------------- | ------------------------- |
| 로직 타일     | `16 x 16 px`               | 바닥, 벽, 충돌 기준       |
| 오버레이      | `32 x 32 px`               | 풀, 흙, 균열, 표면 정보   |
| 중형 오브젝트 | `48 x 48 px`, `64 x 64 px` | 우물, 수레, 장치, 문틀    |
| 랜드마크      | `96 x 96 px` 이상          | 시계탑, 구조선, 대형 수목 |

### 제작 규칙

- 바닥과 벽은 `16 x 16`만으로 끝내지 않는다.
- 기본 타일 위에 `32 x 32` 오버레이를 겹친다.
- 빈 단색 타일 반복을 피한다.
- 장식이 격자를 넘더라도 충돌 박스는 격자를 유지한다.

### 권장 구조 치수

| 요소                  | 권장 크기          |
| --------------------- | ------------------ |
| 기본 바닥 두께        | `1 타일`           |
| 낮은 턱               | `0.5 타일`         |
| 짧은 단차             | 높이 `2 타일`      |
| 문/입구 폭            | `2~3 타일`         |
| 문/입구 높이          | `3~4 타일`         |
| 체크포인트 장치 폭    | `2~3 타일`         |
| 체크포인트 장치 높이  | `4~5 타일`         |
| 화면 내 핵심 랜드마크 | 폭 `4~8 타일` 이상 |

## 5. 플레이어 규격

### 기본 스펙

| 항목                   | 값                               |
| ---------------------- | -------------------------------- |
| 기본 스프라이트 캔버스 | `32 x 48 px`                     |
| 확장 액션 캔버스       | `48 x 48 px`                     |
| 긴 액션 캔버스         | `48 x 64 px`                     |
| 실루엣 점유            | 가로 `26~30 px`, 세로 `44~46 px` |
| 기본 충돌 박스         | `14 x 30 px`                     |
| 발 위치 기준점         | 스프라이트 하단 중앙             |
| 머리 여백              | 상단 `2 px` 내외                 |
| 좌우 여백              | 각 `1~3 px`                      |

### 실루엣 규칙

- 외투
- 벨트 장비
- 등 장치
- 상체 기울기
- 보행 자세

위 5개가 먼저 읽혀야 한다.

### 판정 규칙

| 판정                     | 기준                       |
| ------------------------ | -------------------------- |
| 서기 충돌                | `14 x 30 px`               |
| 상호작용 거리            | 전방 `16 px`, 상하 `14 px` |
| 기본 공격 유효 전방 거리 | `22~28 px`                 |
| 근접 피격 기준폭         | 중심에서 좌우 `9 px`       |
| 발판 허용 오차           | 하강 시 `4 px`             |

### 2차 모션 허용 범위

- 외투 끝단
- 허리 장비
- 머리카락 끝
- 등 장치 또는 등불
- 강공격 후 손목 / 칼끝

가동 범위는 `1~3 px` 이내로 제한한다.

## 6. 마스터 스프라이트 승인 절차

### 작업 순서

1. 전체 모션 시트 제작 전에 `정지 기본 포즈 3안`을 먼저 만든다.
2. 3안 중 1안을 `마스터 스프라이트`로 확정한다.
3. 확정된 마스터 스프라이트 기준으로 `idle` 키포즈를 만든다.
4. `idle` 승인 후에만 이동 / 전투 모션 제작으로 넘어간다.

### 제출 규격

- 기본 배경 없는 PNG
- `100%` 보기 이미지
- `400%` 확대 보기 이미지
- 밝은 배경 위 미리보기 1장
- 어두운 배경 위 미리보기 1장
- 사용 팔레트 표기

### 승인 조건

- 얼굴, 머리, 외투, 손, 발, 장비 포인트가 구분된다.
- 정지 상태에서 실루엣만 봐도 캐릭터 축이 읽힌다.
- 외투가 몸통과 다리를 완전히 덮지 않는다.
- 색 분리가 충분해 몸통, 팔, 다리 경계가 읽힌다.
- 확대했을 때 잡선보다 덩어리와 클러스터가 먼저 보인다.

### 금지 규칙

- 마스터 스프라이트 승인 전 전체 모션 시트 제작 금지
- 정지 포즈 1안만 제출하고 바로 진행하는 방식 금지
- 확대 보기 없이 결과물 제출 금지

## 7. NPC / 적 / 오브젝트 비율

### 캐릭터

| 분류           | 권장 스프라이트                | 플레이어 대비 |
| -------------- | ------------------------------ | ------------- |
| 일반 NPC       | `28 x 44 px` 또는 `32 x 48 px` | `0.95~1.0x`   |
| 동행 주요 NPC  | `32 x 48 px`                   | `1.0x`        |
| 중장비형 NPC   | `40 x 56 px`                   | `1.15x`       |
| 일반 소형 적   | `24 x 24 ~ 32 x 32 px`         | `0.55~0.7x`   |
| 일반 인간형 적 | `32 x 48 px`                   | `1.0x`        |
| 엘리트 적      | `48 x 64 px`                   | `1.3x`        |
| 대형 적        | `64 x 64 ~ 96 x 80 px`         | `1.6x+`       |
| 보스 핵심 부위 | 높이 `96 px` 이상              | `2.0x+`       |

### 오브젝트

| 분류               | 권장 크기                                |
| ------------------ | ---------------------------------------- |
| 조사 오브젝트 소형 | `16 x 16 px`, `16 x 32 px`               |
| 조사 오브젝트 중형 | `32 x 32 px`, `32 x 48 px`               |
| 핵심 상호작용 장치 | `48 x 64 px`                             |
| 엄폐 / 장애물      | `32 x 32 px`, `48 x 32 px`, `48 x 48 px` |
| 배경 문 / 게이트   | `48 x 64 px` 이상                        |
| 배경 랜드마크      | `96 x 96 px` 이상                        |

### 적용 규칙

- 일반 시민 NPC도 작아 보이지 않게 한다.
- 주요 동료는 플레이어와 같은 해상도 계열로 제작한다.
- 엘리트 적은 크기, 덩어리, 모션 길이를 함께 키운다.
- 핵심 상호작용 장치는 플레이어보다 약간 크게 만든다.

## 8. 배경 레이어 기준

### 기본 레이어

1. 원경: 하늘, 먼 구조물, 큰 실루엣
2. 중경: 벽, 나무, 건물, 기둥
3. 플레이층: 바닥, 오브젝트, NPC, 적
4. 전경: 풀, 천, 잔가지, 먼지, 난간 일부

### 장면 밀도 규칙

- 화면마다 중형 오브젝트 `2~4개` 배치
- 바닥 가장자리에 식생, 잔해, 균열, 파편 배치
- 수평 진행 구간에 수직형 실루엣 반복 배치
- 원경은 그라디언트 단독 사용 금지

### 타일셋 최소 구성

- 평지 바닥
- 바닥 끝단
- 안쪽 코너
- 바깥 코너
- 균열 / 파손 버전
- 식생 또는 먼지 오버레이
- 벽 상단 마감
- 난간 또는 경계 구조
- `32 x 32` 이상 장식 패치 4종 이상

### 파랄랙스 권장값

| 레이어                | 이동 비율  |
| --------------------- | ---------- |
| 원경 하늘 / 먼 구름   | `0.1~0.18` |
| 먼 구조물             | `0.2~0.3`  |
| 중경 벽 / 수목        | `0.45~0.6` |
| 전경 식생 / 천 / 기둥 | `0.8~0.95` |

## 9. 팔레트와 셰이딩

### 캐릭터 팔레트

- 핵심 팔레트 `12~18색`
- 의상 재질은 최소 3단계 명암
- 발광 포인트는 재질색과 분리

### 배경 팔레트

- 지역당 핵심 색상군 `24~36색`
- 흙, 목재, 금속, 식생, 천은 색온도 차이를 둔다.
- 같은 갈색, 같은 회색으로 재질을 통합하지 않는다.

### 셰이딩 규칙

- 검은 외곽선 고정 사용 금지
- 배경 밝기에 따라 유색 외곽선 허용
- 얼굴과 손은 단순 면 우선
- 디테일은 장비와 옷자락에 우선 배분
- 배경은 타일 디테일보다는 덩어리와 명암 리듬을 우선

### 금지 규칙

- 단순 확대 후 샤프닝 금지
- 모든 재질 2단계 명암 처리 금지
- 식생, 흙, 나무 동일 외곽선 처리 금지
- FX로 본체 실루엣 가리기 금지

## 10. 카메라 기준

### 기본 카메라

- 기본 시야: 내부 기준 `320 x 180 px` / 출력 기준 `1280 x 720 px`
- 플레이어 위치: 진행 방향 기준 `38~42%`
- 수직 위치: 하단 기준 `38~42%`
- 상하 시프트: `8~12 px`

### 장면 구성 규칙

- 플레이어 주변 `3 타일` 안은 읽기 쉬워야 한다.
- 그 바깥 `4~10 타일` 구간에서 배경 밀도를 채운다.
- 입력 방해 요소를 플레이어 발밑 주변에 과도하게 두지 않는다.

### 전투 카메라

- 일반 교전은 기본 시야 유지
- 엘리트 / 보스전은 최대 `110%` 줌아웃
- 확대 범위는 `352 x 198 px` 이내
- 전투 구역 폭은 기본 시야의 `1.2배` 이내

## 11. 스프라이트 품질 체크리스트

### 실루엣 체크

- 축소된 상태에서도 머리와 몸통 방향이 구분된다.
- 양팔 위치가 몸통에 묻히지 않는다.
- 외투와 다리가 한 덩어리로 뭉개지지 않는다.
- 손이나 무기 손잡이 같은 행동 포인트가 최소 1곳 이상 읽힌다.

### 색 분리 체크

- 피부, 머리, 의상 기본색, 암부, 악센트색이 분리된다.
- 의상 전체가 단일 청색 덩어리로 보이지 않는다.
- 몸통과 다리의 명도 차이가 확보된다.
- 팔 안쪽과 몸통이 같은 색으로 붙어 보이지 않는다.

### 픽셀 처리 체크

- `1 px` 잡선 남발 금지
- 큰 덩어리와 클러스터를 우선 정리
- 곡선을 표현할 때 계단이 과하게 튀지 않게 정리
- 의미 없는 단독 픽셀과 노이즈 셰이딩 금지

### 제출 전 체크

- `100%` 보기에서 읽힌다.
- `400%` 보기에서 구조가 무너지지 않는다.
- 밝은 배경 위에서 실루엣이 죽지 않는다.
- 어두운 배경 위에서 외곽이 사라지지 않는다.

### 제작 규칙

- `준비 -> 타격 -> 후속 -> 정리` 단계를 분리한다.
- 걷기와 달리기는 손발보다 몸축 이동이 먼저 읽혀야 한다.
- 외투와 장비는 본동작보다 약간 늦게 따라오게 한다.
- 점프와 착지는 무릎 압축과 발목 반동이 보여야 한다.

### 축소 우선순위

1. `idle`, `walk`, `run`, `attack_1~3`, `heavy_attack` 유지
2. 변형 모션 축소
3. 배경 루프 축소
4. 플레이어 핵심 프레임 수는 마지막에만 축소

- 발광 색은 캐릭터 / 지역 팔레트와 충돌하지 않게 관리한다.

## 17. 최종 고정안

1. 월드 로직은 `16 x 16 px` 타일 기준으로 유지한다.
2. 기준 제작 / 검수 해상도는 `1280 x 720 px` (`720p`)로 고정한다.
3. 내부 월드 렌더링은 `320 x 180 px`를 사용한다.
4. 플레이어 기본 비주얼은 `32 x 48 px`를 사용한다.
5. 플레이어 확장 액션은 `48 x 48 px` 이상을 사용한다.
6. 플레이어 충돌은 `14 x 30 px`를 기본으로 한다.
7. 플레이어 모션 제작 전 `정지 기본 포즈 3안 -> 마스터 스프라이트 승인 -> idle 승인` 순서를 지킨다.
8. 배경은 `기본 타일 + 오버레이 + 중형 오브젝트 + 원경 레이어` 구조로 제작한다.
9. 걷기 `8`, 달리기 `10`, 공격 `6~8` 프레임을 기본 기준으로 한다.
10. 탐험 모션은 `10~12 FPS`, 전투 모션은 `12~16 FPS` 체감으로 제작한다.
11. 디테일 배분 우선순위는 `실루엣 -> 포즈 차이 -> 2차 모션 -> FX`다.
