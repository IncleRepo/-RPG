# 코어 게임 시스템 설계

## 문서 목적

이 문서는 `src/content/scenario/`와 `src/content/lore/` 문서를
실제 브라우저 기반 2D RPG 구현 단위로 분해한 기준안이다.

목표는 다음 세 가지다.

- 시나리오 문서의 장면을 `탐험`, `전투`, `대화`, `퀘스트`, `보상` 흐름으로 바로 옮길 수 있게 한다.
- 구현자가 기능 우선순위와 상태 전이를 빠르게 이해할 수 있게 한다.
- 세계관의 핵심 규칙인 `강한 미래 = 더 큰 대가`를 게임 플레이에 일관되게 반영한다.

## 설계 기준

- 장르 기준: `실시간 필드 탐험 + 인스턴스형 실시간 전투` 구조의 스토리 중심 액션 RPG
- 플레이타임 기준: 메인 스토리 약 8시간
- 플랫폼 기준: 데스크톱 브라우저 우선, 이후 모바일 UI 확장 가능 구조
- 시점 기준: 2D 탑다운 필드 탐험, 전투는 전용 전투 룸 또는 전투 전용 배경으로 전환
- 성장 기준: 장시간 파밍보다 `챕터 진행`, `종편 회수`, `관계 변화` 중심의 마일스톤 성장

이 문서는 `오픈 월드`가 아니라
`허브 + 지역 세트 + 장면 단위 이벤트` 구조를 기본값으로 둔다.
현재 프로젝트 범위와 시나리오 밀도를 고려하면 이 방식이 구현과 연출 제어에 가장 유리하다.

## 핵심 플레이 판타지

플레이어는 `여명 인양사`로서 다음 네 가지를 반복 체험해야 한다.

1. 균열 지역을 조사하며 아직 도착하지 않은 정보와 감정을 읽는다.
2. 짧은 미래 시연 능력으로 위험을 비껴 가거나 전투 흐름을 바꾼다.
3. 종편과 내일결을 회수하며 더 강한 힘과 더 큰 대가를 동시에 떠안는다.
4. 누구를 살리고 무엇을 포기할지 결정하며 관계와 세계 상태를 바꾼다.

## 전체 게임 루프

### 거시 루프

1. 허브 또는 챕터 시작 지점에서 브리핑과 장비 정비를 한다.
2. 메인 지역으로 진입해 탐험, 조사, NPC 대화, 퀘스트를 수행한다.
3. 필수 이벤트를 통해 전투, 퍼즐, 추격, 구조전 중 하나 이상을 해결한다.
4. 종편 또는 핵심 단서를 회수하고 챕터 보상과 관계 변화를 정산한다.
5. 허브 또는 다음 지역으로 이동해 새 스킬, 장비, 대화를 해금한다.

### 미시 루프

1. 이동한다.
2. 조사 가능한 오브젝트나 NPC를 찾는다.
3. `새벽 청취` 또는 상호작용으로 단서를 읽는다.
4. 전투 또는 지역 기믹을 해결한다.
5. 보상, 체크포인트, 대사 변화를 획득한다.

## 챕터별 시스템 역할

### 프롤로그

- 허브 이동, 기본 상호작용, 조사 포인트 읽기
- 첫 전투 튜토리얼
- `선행 발자국`, `미도착 상처` 기본 사용법 학습
- 항구형 허브와 시계탑형 소형 던전의 2단 구조 검증

### 챕터 1

- 구조 요청, 방어형 전투, 분기형 필드 목표
- 사막 지역 시야 방해, 구조 대상 보호, 이동 동선 판단
- 라오 템과의 생존 윤리 충돌을 퀘스트 선택으로 반영

### 챕터 2

- 입체 맵, 역중력, 시간 지연 문 개폐
- 시민 보호전과 코어실 전투처럼 연속된 전투 이벤트 처리
- 질서 유지형 세력과의 협상 결과를 지역 상태에 반영

### 챕터 3

- 대화 밀도 증가
- 감정 조사, 기억 퍼즐, 자기 실루엣 이벤트
- `기억 닻`을 활용한 되감기형 기믹과 선택 전 상태 고정

### 엔딩

- 붕괴형 이동 전투
- 종편 7개 결과 반영
- 최종 선택과 엔딩 분기 처리

## 상태 전이 설계

전체 게임은 명확한 상태 머신으로 관리한다.

### 최상위 상태

- `boot`: 리소스 로드, 세이브 슬롯 확인
- `title`: 시작, 이어하기, 문서/설정 진입
- `hub`: 브리핑, 대화, 정비, 출항 선택
- `field`: 이동, 조사, 수집, 필드 이벤트
- `dialogue`: 컷신, 선택지, 관계 변화
- `combat`: 인스턴스형 전투 진행
- `result`: 전투/퀘스트 보상, 종편 획득, 지역 상태 변경
- `pause`: 인벤토리, 설정, 저장
- `game-over`: 체크포인트 복귀 또는 타이틀 이동

### 전이 규칙

- `field -> dialogue`: NPC와 상호작용하거나 장면 트리거에 진입했을 때
- `field -> combat`: 적 심볼 접촉, 강제 연출, 보스 트리거 발생 시
- `dialogue -> field`: 일반 대화 종료 시
- `dialogue -> combat`: 대화 직후 전투 연출이 이어질 때
- `combat -> result`: 승리, 철수 성공, 방어 성공 조건 달성 시
- `combat -> game-over`: 파티 전멸 또는 핵심 보호 목표 실패 시
- `result -> field`: 같은 지역을 계속 탐험할 때
- `result -> hub`: 챕터 종료 또는 허브 귀환 연출 시

## 탐험 시스템

### 맵 구조

맵은 다음 네 타입으로 분리한다.

- `hub-map`: 미라진 같은 안전 구역, NPC 대화와 정비 중심
- `field-map`: 자유 이동과 조사 중심의 일반 탐험 지역
- `event-room`: 컷신, 퍼즐, 구조 이벤트가 발생하는 장면형 공간
- `combat-room`: 전투 전용 배경 또는 작은 투기장 단위 공간

각 챕터는 최소 아래 구성을 가진다.

- 메인 필드 1종 이상
- 변형 필드 1종 이상
- 전투 룸 2종 이상
- 보스 룸 1종
- 감정 대화 전용 공간 1종

### 필드 상호작용 오브젝트

- `echo-node`: 미래 잔향을 읽는 조사 포인트
- `loot-node`: 내일결, 회복 자원, 문서 파편 획득
- `gate-device`: 열쇠, 시간 조건, 퍼즐 해결 후 개방
- `hazard-zone`: 모래폭풍, 역중력, 붕괴 구역처럼 지속 위협을 주는 지역
- `rescue-target`: 보호, 운반, 동행 이벤트가 걸린 NPC 또는 장치
- `anchor-point`: 저장, 회복, 장비 교체, 체크포인트 갱신

### 필드 규칙

- 탐험 지역은 미니맵보다 `현재 목표`, `근처 상호작용`, `진입 가능한 출구`를 명확히 보여 주는 방식이 우선이다.
- 조사 포인트는 단순 텍스트 로그가 아니라 `대사`, `효과`, `다음 목표` 중 하나를 반드시 남긴다.
- 동일 지역 재방문 시 NPC 대사와 오브젝트 상태가 챕터 진행도에 따라 바뀌어야 한다.
- 점프 플랫폼 액션보다 `경로 선택`, `장치 조작`, `타이밍 회피`에 무게를 둔다.

## 전투 시스템

### 기본 전투 구조

전투는 `실시간 액션`을 기본으로 하되,
필드 탐험과 연출 제어를 위해 `전용 전투 상태`로 분리한다.

기본 흐름은 다음 순서다.

1. 전투 진입 연출
2. 플레이어와 적의 배치
3. 실시간 공격, 회피, 능력 사용
4. 보스 패턴 또는 보호 목표 처리
5. 승패 판정과 보상 정산

### 플레이어 기본 액션

- `기본 공격`: 짧은 연속 공격, 공명력 회수의 기본 수단
- `강공격`: 경직과 약점 파괴에 특화, 후딜이 길다
- `회피`: 방향 입력 기반 짧은 무적 대시
- `상호작용`: 전투 오브젝트, 보호 대상, 기믹 장치 활성화
- `능력 슬롯`: 여명 인양사 기술 2개, 종편 기술 2개를 장착
- `아이템 슬롯`: 회복제, 버프, 전투용 도구 사용

### 전투 자원

- `HP`: 생존 자원
- `공명력`: 기술 사용 자원, 기본 공격과 완벽 회피로 회복
- `기억 마모`: 강한 능력을 쓸수록 누적되는 페널티 자원
- `과부하 단계`: 기억 마모가 한계를 넘으면 전투 중 디버프 또는 대사 왜곡 발생

### 기억 마모 규칙

- 약한 기술은 공명력만 사용한다.
- 강한 기술과 종편 공명은 `공명력 + 기억 마모`를 함께 요구한다.
- 기억 마모가 높을수록 화면 흔들림, 청취 오류, 스킬 재사용 시간 증가 같은 부작용을 준다.
- 챕터 클리어, 휴식, 특정 NPC 이벤트로만 크게 회복된다.

### 여명 인양사 핵심 능력

- `새벽 청취`: 필드에서는 숨은 단서와 거짓 대사를 읽고, 전투에서는 적의 약점 창을 잠깐 노출한다.
- `선행 발자국`: 잠깐 뒤의 위치를 먼저 지정해 짧은 순간이동처럼 활용한다.
- `미도착 상처`: 받은 피해 일부를 지연시켜 즉사 위험을 줄인다.
- `기억 닻`: 짧은 시간 전 상태를 고정해 특정 위치나 상황으로 되돌아간다.
- `종편 공명`: 챕터 후반 해금되는 고위 기술, 각 종편 성질에 따라 장면 규칙을 바꾼다.

### 적 분류

- `균열 생물`: 빠른 템포, 패턴 학습용 기본 적
- `장치 수호체`: 기믹과 약점 파괴 중심
- `인간 세력`: 방패, 사격, 제압 기술을 사용하는 전술형 적
- `보스`: 장면 규칙이 함께 붙는 대형 적

### 전투 유형

- `일반 교전`: 소수 적 처치
- `방어전`: 오브젝트 또는 시민 보호
- `추격전`: 일정 거리 이상 뒤처지지 않고 이동
- `연속전`: 짧은 휴식 후 바로 다음 웨이브 진행
- `보스전`: 다중 패턴과 약점 노출 기믹 포함

### 승패 규칙

- 일반 교전은 모든 적 처치 시 승리
- 방어전은 제한 시간 종료 또는 목표 지점 도달 시 승리
- 추격전은 목표 거리 유지 및 최종 교전 진입 시 성공
- 플레이어 HP 0 또는 핵심 보호 대상 사망 시 실패
- 실패 시 마지막 `anchor-point`로 복귀하되, 대화 분기는 유지하고 소모품 일부만 복구한다

## 퀘스트 시스템

### 퀘스트 분류

- `main-quest`: 챕터 진행 필수
- `area-quest`: 지역 상황과 세력 갈등을 보강
- `rescue-quest`: 구조와 방어 중심
- `echo-quest`: 잔향 조사와 감정 회수 중심
- `bond-scene`: 동료 또는 핵심 인물과의 관계 이벤트

### 퀘스트 구조 규칙

- 모든 메인 퀘스트는 `목표`, `진행 조건`, `완료 조건`, `실패 조건`, `후속 상태`를 가져야 한다.
- 장면 전환이 있는 퀘스트는 `대화 -> 탐험 -> 전투 -> 보상` 중 어떤 단계를 포함하는지 명시한다.
- 선택지는 분기 수를 늘리기보다 `보상`, `관계`, `지역 반응`을 바꾸는 방향으로 사용한다.
- 챕터 종료에 영향을 주는 선택은 최대 1개만 배치하고, 나머지는 누적형 반응으로 관리한다.

### 관계 시스템

- 주요 관리 대상은 `사야 렌`, `라오 템`, `유라 베인` 3인이다.
- 관계 수치는 단순 호감도보다 `신뢰`, `동의`, `상실 위험` 세 태그로 관리한다.
- 선택 결과는 엔딩 분기보다 `동행 대사`, `전투 지원 효과`, `개인 이벤트 해금`에 먼저 반영한다.

## 성장과 보상 시스템

### 성장 원칙

- 적 처치 경험치 누적형 레벨업은 보조 수단으로만 사용한다.
- 핵심 성장은 `종편 회수`, `퀘스트 완료`, `관계 이벤트`, `장비 해금`으로 이뤄진다.
- 메인 시나리오 8시간 기준으로 파밍 없이도 필수 성장이 충분해야 한다.

### 성장 축

- `항해 등급`: 챕터 완료 시 기본 능력치와 슬롯 해금
- `종편 슬롯`: 종편 개수에 따라 강력한 공명 기술과 패시브 해금
- `기록 장비`: 무기보다 도구 성격이 강한 장비 세트, 공격 방식과 보조 효과 변경
- `내일결 튜닝`: 청람결, 금백결, 자홍결, 흑은결을 장착해 스킬 속성과 부가 효과 변경
- `관계 보너스`: 주요 인물과의 신뢰가 높을수록 특정 장면과 보조 효과 해금

### 보상 종류

- `내일결 파편`: 소모성 제작과 튜닝 재화
- `항로 기록`: 퀘스트 로그 확장, 숨은 지역 해금 단서
- `장비 설계도`: 새 도구와 패시브 개방
- `종편`: 서사와 시스템을 동시에 확장하는 최상위 보상

## 퍼즐과 지역 기믹

### 공통 원칙

- 퍼즐은 수학 문제보다 세계 규칙 이해를 요구해야 한다.
- 조사와 대화에서 얻은 정보가 퍼즐 힌트로 이어져야 한다.
- 실패해도 즉시 재시도 가능해야 하며, 긴 역주행은 최소화한다.

### 챕터별 기믹 기준

- 프롤로그: 동일 대사의 반복, 먼저 열린 문, 미래 잔향 읽기
- 챕터 1: 모래폭풍 시야 제한, 구조 경로 우선순위, 사막 잔향 추적
- 챕터 2: 역중력 발판, 시간 지연 문, 층 전환형 이동
- 챕터 3: 감정 열매 회수, 선택되지 않은 역사 읽기, 실루엣 추적
- 엔딩: 붕괴 타이머, 이동 전투, 종편 공명으로 장면 규칙 변경

## UI 및 HUD 기준

### 필드 HUD

- 현재 챕터와 지역명
- 플레이어 HP와 공명력
- 현재 목표 1줄
- 상호작용 프롬프트
- 사용 가능한 여명 기술 슬롯
- 기억 마모 경고

### 전투 HUD

- 플레이어 HP, 공명력, 기억 마모
- 장착 능력 4칸과 재사용 대기시간
- 적 보스 HP 또는 보호 목표 상태
- 전투 목표 텍스트
- 상태 이상과 과부하 경고

### 대화 UI

- 화자명, 초상 또는 실루엣
- 스킵, 로그 다시 보기, 자동 진행
- 선택지 최대 3개
- 관계 영향이 큰 선택은 연출로만 암시하고 수치 자체는 직접 노출하지 않는다

## 저장 데이터 기준

세이브 데이터는 최소 아래 정보를 가져야 한다.

- 현재 챕터, 장면, 체크포인트
- 플레이어 능력치와 장착 기술
- 종편 회수 상태
- 퀘스트 상태와 주요 선택 플래그
- 관계 태그 상태
- 인벤토리와 내일결 튜닝 정보
- 지역 오브젝트 해제 상태
- 누적 플레이 시간

자동 저장 시점은 다음이 기본이다.

- 메인 대화 종료 직후
- 전투 승리 직후
- 종편 획득 직후
- 허브 복귀 직후
- 수동 저장 포인트 사용 시

## 데이터와 코드 구조 제안

현재 저장소 구조를 기준으로 아래 모듈 분리를 권장한다.

- `src/scripts/core/`: 상태 머신, 루프, 세이브 로더
- `src/scripts/field/`: 맵, 이동, 상호작용, 이벤트 트리거
- `src/scripts/combat/`: 전투 상태, 액터, 스킬, 판정
- `src/scripts/quest/`: 퀘스트 정의, 진행도, 선택 플래그
- `src/scripts/content/`: 챕터 데이터, 적 데이터, 장비 데이터
- `src/scripts/ui/`: HUD, 대화창, 인벤토리, 결과 화면

콘텐츠 데이터는 하드코딩된 조건문보다
`chapterId`, `sceneId`, `objectiveId`, `rewardId` 중심의 선언형 구조를 우선한다.

## 구현 우선순위

### 1차 구현

- 플레이어 이동
- 필드 상호작용
- 대화창
- 체크포인트 저장
- 전투 진입과 종료

### 2차 구현

- 보스전 1종

- 관계 변화와 보조 효과
- 내일결 튜닝
- 종편 공명 기술

### 4차 구현

- 엔딩 분기 반영
- 재방문 대사 변화
- 세부 밸런싱
- 접근성 옵션과 모바일 HUD 조정

## MVP 완료 기준

아래 조건을 만족하면 코어 시스템 MVP로 본다.

- 프롤로그 한 챕터를 처음부터 끝까지 플레이할 수 있다.
- 이동, 조사, 대화, 기본 전투, 체크포인트 저장이 모두 연결된다.
- `선행 발자국`, `미도착 상처`가 실제 전투와 탐험에서 모두 의미 있게 작동한다.
- 전투 실패 후 체크포인트 복귀와 재도전 흐름이 자연스럽다.
- 퀘스트 상태와 장면 플래그가 저장/불러오기 이후에도 유지된다.

## 후속 문서 연결

- `src/content/scenario/00-overview.md`는 챕터 단위 장면 기준 문서다.
- `src/content/scenario/01-prologue.md`부터 `05-ending.md`까지는 실제 장면과 전투 배치 기준 문서다.
- `src/content/lore/world-rules.md`는 자원, 대가, 지역 기믹 설계의 근거 문서다.
- `src/content/lore/characters.md`는 관계 이벤트와 동행 보조 효과 설계의 기준 문서다.

이 문서를 기준으로 다음 단계에서는
`프롤로그 플레이어블 명세`, `전투 프로토타입 명세`, `퀘스트 데이터 포맷`을 세부 문서로 분리하는 것이 적절하다.
